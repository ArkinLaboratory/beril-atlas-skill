## PLUGIN_GUIDE.md head redirect banner

Per the 2026-05-05 inter-team direction, `PLUGIN_GUIDE` is no longer
the uniform pattern across the four skills — the agreed pattern is
TUTORIAL + HUB_INSTALL + CONFIGURE. Atlas keeps its PLUGIN_GUIDE as
the comprehensive operator reference (CLI surface + flag economy is
broad enough to warrant a single grep-able landing page) but the
new banner directs:

- New-to-the-suite traffic → cross-skill PARTICIPANT-RUNBOOK.
- Quickstart traffic → TUTORIAL.md.
- Provider-setup traffic → CONFIGURE.md.
- Depth traffic (every flag, every error class) → this document.

Owns the kept-as-comprehensive-reference choice with rationale, so
future maintainers know why this skill kept its PLUGIN_GUIDE while
the others retire theirs.

## TUTORIAL.md head deference banner

Adds an explicit reading-order pointer + drift-tracking line:
"Shared content tracked in PARTICIPANT-RUNBOOK; this document
last-checked against it 2026-05-05. If you spot drift between them,
the runbook wins for shared content; flag the drift to the atlas
maintainer."

Concrete drift-tracking discipline: every release that touches
TUTORIAL bumps the last-checked date OR explicitly notes that the
runbook hasn't moved. Lightweight enough to actually happen; gives
future readers a clear signal about content currency.
